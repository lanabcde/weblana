BABA I : Interpreter dan Compiler

Tujuan Pembelajaran
  • Tujuan 1
  • Tujuan 2

Pemrograman komputer adalah tindakan menyusun elemen-elemen bahasa pemrograman yang dipilih secara berurutan agar menghasilkan efek yang diinginkan. Efek tersebut dapat berbeda dalam setiap kasus tertentu - tergantung pada imajinasi, pengetahuan, dan pengalaman programmer.

  1. Secara alfabetis - program harus ditulis dalam aksara yang dapat dikenali, seperti Romawi, Kiril, dll.
  2. secara leksikal - setiap bahasa pemrograman memiliki kamusnya sendiri dan Anda perlu menguasainya; untungnya, itu jauh lebih sederhana dan lebih kecil dari kamus bahasa alami mana pun
  3. Secara sintaksis - setiap bahasa memiliki aturannya sendiri dan aturan tersebut harus dipatuhi
  4. Secara semantis - program harus masuk akal.

Sayangnya, seorang programmer juga bisa membuat kesalahan dengan keempat hal di atas. Setiap hal tersebut dapat menyebabkan program menjadi sepenuhnya tidak berguna. Anggaplah kita telah berhasil menulis sebuah program. Bagaimana kita meyakinkan komputer untuk menjalankannya? kita harus mengubah program menjadi bahasa mesin. Untungnya, terjemahan dapat dilakukan oleh komputer itu sendiri, membuat seluruh proses menjadi cepat dan efisien. Ada dua cara yang untuk mengubah program dari bahasa permograman tingkat tinggi menjadi bahasa mesin :

Compiling

Source Code diterjemahkan sekali (namun, tindakan ini harus diulang setiap kali kita memodifikasi kode sumber) dengan mendapatkan file (misalnya, file .exe jika kode dimaksudkan untuk dijalankan di bawah MS Windows) yang berisi kode mesin; sekarang Anda dapat mendistribusikan file tersebut ke seluruh dunia; program yang melakukan terjemahan ini disebut compiler atau translator

Interpreting

Kita (atau pengguna kode apa pun) dapat menerjemahkan Source code setiap kali harus dijalankan; program yang melakukan transformasi semacam ini disebut interpreter, karena ia menginterpretasi kode setiap kali dimaksudkan untuk dieksekusi; ini juga berarti bahwa kita tidak dapat hanya mendistribusikan kode sumber apa adanya, karena pengguna akhir juga memerlukan interpreter untuk mengeksekusinya. Karena beberapa alasan yang sangat mendasar, bahasa pemrograman tingkat tinggi tertentu dirancang untuk termasuk ke dalam salah satu dari dua kategori ini. Hanya ada sedikit bahasa yang dapat diterjemahkan dan dikompilasi. Biasanya, bahasa pemrograman diproyeksikan dengan faktor ini dalam pikiran konstrukturnya - apakah itu akan dikompilasi atau diinterpretasikan?
